# V1 netcode and wire protocol

Crown Tag V1 uses a fixed **20 Hz** authoritative tick in the Arena Durable Object (`dt = 1/20`). After Matchmaker HTTP join, the client holds one **Hibernation WebSocket** to that Arena. The client sends sequenced **input** messages (move axes, jump, sprint, hit intent). The server broadcasts a **full snapshot** every tick (≤12 Fighters — deltas deferred) and also emits discrete **events** for Hit, Steal, Claim, Join, and Leave so presentation juice does not depend on client-side snapshot diffs. The client runs `@crowntag/sim` for **local movement prediction** and reconciles by rewind-and-replay against authoritative snapshots; Steal, Stun, and Knockback are **not** fully client-predicted — they apply from server events/snapshots and force reconcile on the affected Fighter. Message shapes live in `@crowntag/protocol`; this ADR locks the contract, not the TypeScript.
