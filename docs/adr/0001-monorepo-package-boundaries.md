# Monorepo package boundaries for V1

Crown Tag V1 is a pnpm monorepo with content / sim / presentation separation, plus an explicit wire package. `@crowntag/content` holds tweakable data only; `@crowntag/sim` owns authoritative Arena rules and runs in the Arena Durable Object and in tests (no Three.js, DOM, or Cloudflare APIs); `@crowntag/protocol` holds shared command/event/snapshot message types as the client↔worker seam; `apps/client` is Three.js presentation + prediction; `apps/worker` hosts the Worker, Matchmaker DO, Arena DO, and serves the client build as static assets. We chose a separate protocol package over folding wire types into sim so the netcode contract can evolve without implying sim API changes, and split `apps/client` + `apps/worker` rather than one colocated app so presentation stays removable from the deployable host.
